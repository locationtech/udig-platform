%resolvers.name

Identifier:
net.refractions.udig.catalog.resolvers

Since:
[Enter the first release in which this extension point appears.]

Description:
Allows additional content types to be made available through geospatial resources handles. This extention point is used to provide the following: 1. Additional content through use of metadata: - prior knowledge: ie. makeing shape.sld file available as a Style.class - Making a CSW 2.0 association query for a related information Style.class 2. Making additional Java API available In practice this is analagous to the org.eclipse.runtime.adapters extension point except that it applies to net.refractions.udig.catalog.IResolve objects instead of IAdaptable objects.

Configuration Markup:

<!ELEMENT extension (factory+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>

The resolves extension point. Allows IResolves to be extended via the extension point mechanism.


  • id - id of the extension
  • name - name of the extension

<!ELEMENT factory (resolve+)>

<!ATTLIST factory

resolveableType CDATA #IMPLIED

class           CDATA #IMPLIED

>

Declares a factory that is used by an IResolve to find what classes it/they can resolve to. The canResolve method on the IResolveAdapterFactory will not be called. The canResolve will rely on the xml.


  • resolveableType - The fully qualified name of a class (typically implementing IResolve) that this factory provides resolvers for.
  • class - The fully qualified name of the adapter factory class. Must implement net.refractions.udig.catalog.IResolveAdapterFactory

<!ELEMENT resolve EMPTY>

<!ATTLIST resolve

type CDATA #IMPLIED

>

Declares the class that the factory can resolve and IResolve to.


  • type - The fully qualified name of a Java class or interface that this factory can adapt to.
