%friendly.name

Identifier:
net.refractions.udig.catalog.friendly

Since:
uDig 1.1

Description:
Allows additional associations be be discovered between peer geospatial resources (known as friends).

Configuration Markup:

<!ELEMENT extension (friend)*>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT friend EMPTY>

<!ATTLIST friend

class   CDATA #REQUIRED

source  CDATA #REQUIRED

contain CDATA #IMPLIED

>

  • class - Class responsible for examining the source and checking in the catalog to see if it has any known friends.
  • source - Used to perform an "instanceof" check to ensure friend relationship is limited to indicated class.
  • contain - Used to limit processing to resources that contain the indicated text. You can use this to only engage on a geoserver wms server by supplying the string "geoserver/wms"; similar tricks can be used to focus on mapserver or geowebcache servers.

Examples:
This extension point can be used to build up additional associations into the catalog reflecting knoweldege that is "well known". Normally associations are built up by asking a CatalogService; or by checking the various GetCapabilities information published by each service and noticing when two services refer to each other. This extension point offers an alternative; you may be able to "assume" that a geoserver installation and a geowebcache installation on the same machine are serving up the same data. At the very least you can assume that a geoserver wms and a geoserver wfs are serving the same data via two different protocols.

API Information:
This extension point will ask that you implement an instance of IFriend.
