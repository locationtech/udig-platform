Identifier:
net.refractions.udig.catalog.ICatalog

Since:
0.4

Description:
Allows addition of Catalogs to be used by the uDig search facilities. This extention point is processed to geospatial resources for visualization and analysis.

Configuration Markup:

Interface used to advertise additional resources catalogs to the uDig application search mechanisms.



The catalog element is deprecated

Origional ICatalog contirbution which has gotten so specific it can really only be used to contirbute a "local catalog" repository which acts as a repostiory of spatial information that can be searched and also provides facilities to create temporary content.


  • class - ICatalog implementation to be used for searching.
  • name - Name of catalog used a default title when listing possible search location.

Examples:
The local catalog is the most common example; allowing you to search for content recorded known to the application. We would like to see OGC Catalog Serivce supported using this extension point. We had a "google" catalog which has been withdrawn due to the soap search facility being removed.

API Information:
ICatalog is the origional interface - it has gotten more and more specifically tied to the local catalog over time. We invite contributors to use ISearch when advertising the ability to search can discover new geospatial resources; or use IRepository if they have a local repository of content to both search and manage.
