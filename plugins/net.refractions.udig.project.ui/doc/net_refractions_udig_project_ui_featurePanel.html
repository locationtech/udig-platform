Feature Panel

Identifier:
net.refractions.udig.project.ui.featurePanel

Since:
1.2

Description:
Declares a panel used to edit feature content. The panel is setup to work only against specific feature content. The panel is available in the feature view when editing a selected feature (or optionally displayed in a wizard when creating a new feature).

Configuration Markup:

<!ELEMENT extension (featurePanel)*>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT featurePanel (featureType | checkForm)?>

<!ATTLIST featurePanel

name          CDATA #REQUIRED

title         CDATA #IMPLIED

description   CDATA #IMPLIED

id            CDATA #IMPLIED

afterPanel    CDATA #IMPLIED

panel         CDATA #REQUIRED

labelProvider CDATA #IMPLIED

>

A feature panel shown to a user to edit the "current" feature. All communication with the current feature is handled through a FeatureSite ... allowing the panel to be reused in either a wizard or a view.


  • name - The name of the feature panel; displayed to the user as a tab or wizard dialog title. This name should be concise ( read short ) but sufficient to communicate to the user what he or she is working on. This field is translatable using"%key" to indicate which plugin.properties definition to use.
  • title - The title of the current group of fields; or wizard page. This is used along with the description to describe what is happening right now. We will probably supliment this with a class so the title and description can be dynamic and reflect the current state of the feature. This field is translatable using"%key" to indicate which plugin.properties definition to use.
  • description - Combined with the title to describe additional information about the group of fields the user is working on. We will probably supliment this with a class so the title and description can be dynamic and reflect the current state of the feature. This field is translatable using"%key" to indicate which plugin.properties definition to use.
  • id - The unique id of the feature panel; this is used to refer to the panel when describing order.
  • afterPanel - Reference to another featurePanel; used to sort feature panels into an order. This order is used to order the tabs in a feature view; or order pages in a feature wizard.
  • panel - The feature panel to display; please note the panel is displayed with context information provided by name, title and description so you do not need to repeate this information in your panel. Please do use a good layout manager as the panel may be displayed in a horizontal or vertical orientation. it is recommended to break out multiple columns to use horizontal space if avaialble ... with this in mind virtical scrolling will be available; horizontal scrolling is not encouraged.
  • labelProvider - Optional LabelProvider if you would like to control how the current feature is described to the user in the header. By default the "Feature ID" will be shown; you may wish to override this entry if your feature content has a name attribute or similar.

<!ELEMENT featureType (typeName | attribute+)>

FeatureType defines the feature type signature that the editor can edit. If the selected feature is does not have the signature of the defined featureType then the editor will not show up in the "edit with" menu.

Either a typename is defined, which declares that the editor will edit only one type of feature, or a set of attributes. If a set of attributes are declared then the editor will be available for editing any feature with a feature type that contains the set of attributes.



<!ELEMENT attribute EMPTY>

<!ATTLIST attribute

name CDATA #IMPLIED

type CDATA #REQUIRED

>

Defines an attribute in the featureType. Defines a 1-many mapping between an editor and featureTypes. If the name of the attribute is not set then the attribute will match all attributes of the declared type.

See definition of featureType for more information.


  • name - The name of the expected attribute; if left empty we assume you are checking the default geometry.
  • type - This is the Java type of the expected attribute. Examples: - java.lang.String - java.lang.Intenger - com.vividsolutions.jts.geom.MultiPolygon

<!ELEMENT typeName EMPTY>

<!ATTLIST typeName

namespace CDATA #REQUIRED

name      CDATA #REQUIRED

>

They typename of the featureType. Defines a 1-1 mapping between an editor and a featureType. See definition of featureType for more information.


  • namespace - Namespace of the feature type
  • name - name of the featuretype, without the namespace.

<!ELEMENT checkForm EMPTY>

<!ATTLIST checkForm

id    CDATA #REQUIRED

class CDATA #REQUIRED

>

Used to check if the form contribution works on the currently selected editLayer. This facility should be used with care as it will force your contributing plugin to be loaded (indeed checking with featureType is much more efficient).


  • id - id for check used in error reporting and logs
  • class - Check the feature site to ensure your form is able to operate. A full IFeatureSite is provided allowing you to check the GeoResource and/or Layer.

Examples:
CountryEditor from the udig tutorials contains an example implementation: - http://svn.refractions.net/udig/udig/trunk/tutorials/net.refractions.udig.tutorials.featureeditor/

API Information:
FeaturePanel describes a user interface contribution using the same method names as org.eclipse.ui.views that can be contributed to a feature editor or view. The FeaturePanel has access to a FeatureSite offering access to: - IFeaturePanel - EditManager - holding the Transaction - Layer - FeatureStore
