Feature Editor

Identifier:
net.refractions.udig.project.ui.featureEditor

Since:
0.9

Description:
Declares a feature editor. If a feature type is declared for the editor then the editor will only be available for use on features with a matching feature type otherwise (if there is no decalred featureType element) the editor will be available for all features.

Configuration Markup:

<!ELEMENT extension (editor+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>

An extension point where extenders can add custom features editors to udig. Context menus that open when the ActivePage has a feature as part of the selection will have an "Edit Feature" item and an "Edit With" submenu. These new feature editors will begin the "Edit With" menu and if chosen will be also accessible when the "Edit Feature" item is selected(until the time that a different editor is chosen in the "Edit With" menu"


  • point - The id of the extension point.
  • id - The id of the extension.
  • name - A human readable name for the extension.

<!ELEMENT editor (featureType)?>

<!ATTLIST editor

icon       CDATA #IMPLIED

id         CDATA #REQUIRED

name       CDATA #REQUIRED

viewId     CDATA #IMPLIED

dialogPage CDATA #IMPLIED

>

The definition of the editor. Either the viewId or the dialogPage must be specified. If both are specified then the view will be opened.


  • icon - The icon that will be displayed in the "Edit With" menu.
  • id - The unique id of the editor.
  • name - The human readable name of the editor.
  • viewId - If the feature editor is an eclipse view then this attribute will have the id of the view. The view must implement net.refractions.udig.project.ui.IUDIGView
  • dialogPage - If this is defined then a dialog will be opened with the declared IUDIGDialogPage.

<!ELEMENT featureType (attribute+ | typeName)>

FeatureType defines the feature type signature that the editor can edit. If the selected feature is does not have the signature of the defined featureType then the editor will not show up in the "edit with" menu.

Either a typename is defined, which declares that the editor will edit only one type of feature, or a set of attributes. If a set of attributes are declared then the editor will be available for editing any feature with a feature type that contains the set of attributes.



<!ELEMENT attribute EMPTY>

<!ATTLIST attribute

name CDATA #IMPLIED

type CDATA #REQUIRED

>

Defines an attribute in the featureType. Defines a 1-many mapping between an editor and featureTypes. If the name of the attribute is not set then the attribute will match all attributes of the declared type.

See definition of featureType for more information.


  • name -
  • type -

<!ELEMENT typeName EMPTY>

<!ATTLIST typeName

namespace CDATA #REQUIRED

name      CDATA #REQUIRED

>

They typename of the featureType. Defines a 1-1 mapping between an editor and a featureType. See definition of featureType for more information.


  • namespace - Namespace of the feature type
  • name - name of the featuretype, without the namespace.

API Information:
The featureEditor extension point can be used to reference: - Views - Dialogs The featureEditor extension point just adds the option to open the view or dialog when editing a feature.
