Drop Actions

Identifier:
net.refractions.udig.ui.dropActions2

Since:
1.0

Description:
This extension point is used to attach an action to the drop event in a drag n drop transaction.

Configuration Markup:

<!ELEMENT extension (action+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT action (destination+ , acceptedType+ , description?)*>

<!ATTLIST action

class      CDATA #REQUIRED

name       CDATA #REQUIRED

enablesFor CDATA #IMPLIED

>

The action to be performed.


  • class - Class of the action, must be a sublcass of IDropAction.
  • name - Name of the action. Should be a short description of the action purpose. Should be human readable and usable in ui.
  • enablesFor -

    a value indicating the selection count which must be met to enable the action. If specified and the condition is not met, the action is disabled. If omitted, the value is assumed to be 1. The following attribute formats are supported:

    • + - 1 or more items selected
    • multiple, 2+ - 2 or more items selected
    • n - A precise number of items selected. For example: enablesFor=" 4" enables the action only when 4 items are selected

<!ELEMENT destination EMPTY>

<!ATTLIST destination

class CDATA #REQUIRED

adapt (true | false)

>

The target of a drop. Destinations are advertised by the the entitiy enabling drop support. It is usually a user interface construct such as a specific editor or view, however it can be any type of object.


  • class - The destination class.
  • adapt - indicates whether the framework should try to adapt the target object to the type declared in this attribute. If not specified then it is assumed to be false.

<!ELEMENT acceptedType EMPTY>

<!ATTLIST acceptedType

class CDATA #REQUIRED

adapt (true | false)

>

The object being dropped. The type of object is based upon the assiociated transfer object in


  • class - A class that this dropAction knows how to handle
  • adapt - indicates whether the framework should try to adapt the dropped object to the type declared in this attribute. If not specified then it is assumed to be false.

<!ELEMENT description (#PCDATA)>

Describes the action
