Boundary

Identifier:
net.refractions.udig.ui.boundary

Since:
1.3.0

Description:
Define a new boundary to be used application wide as a limit. The use of the current boundary is context specific examples include as a target for Zoom To Extent or as a filter when displaying features in a TableView. Use of the boundary is optional and placed under use control; the default value is considered to be All resulting in no boundary being defined.

Configuration Markup:

<!ELEMENT extension (boundary*)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>

  • id - identifier of boundary contribution used in log messages
  • name - Human readible name of Boundary Contribution may be used in displayed error messages

<!ELEMENT boundary EMPTY>

<!ATTLIST boundary

id    CDATA #REQUIRED

name  CDATA #REQUIRED

class CDATA #REQUIRED

page  CDATA #IMPLIED

>

Boundary used as an optional limit for search, feature display and zoom to extent.


  • id - machine readible identifier suitable as a reference for logging or persistence.
  • name - Boundary name displayed to a user in for example the BoundaryView.
  • class - Implementation of IBoundaryStrategy used to supply a ReferencedEnvelope to any operations that require it.
  • page - Optional Page to be displayed in the Boundary View (or as a pop up) if the user wishes to configure the details of the current boundary. We make use of IPageBookViewPage although the abstract class Page is a recommended starting place.

Examples:
We have a couple good examples if you would like to review some sample code before getting started: BoundaryStrategyScreen, BoundaryStrategyMapCrs and BoundaryStrategyAll. To use the current boundary you can make use of the workbench BoundaryService as follows: 

IBoundaryService boundaryService = PlatformGIS.getBoundaryService();
ReferencedEnvelope extent = boundaryService.getExtent();

API Information:
Contributors are asked to porvide an implementation of IBoundaryStrategy; as an option they may also provide a Page for futher confiuration options.

Supplied Implementation:
Please note the use of a Page is optional; if a page is not provided the user will simply be able to choose from a list; and the Page area will be filled in with a short description.
