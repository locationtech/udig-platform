Description:
An extension point for Style Layer Desriptor (SLD) style content.

Configuration Markup:

<!ELEMENT extension (StyleEditorPage)*>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>

An extension point for adding ui components used to edit a component of an SLD.



<!ELEMENT StyleEditorPage EMPTY>

<!ATTLIST StyleEditorPage

id          CDATA #REQUIRED

label       CDATA #REQUIRED

class       CDATA #REQUIRED

contentType CDATA #REQUIRED

category    CDATA #IMPLIED

requires    CDATA #IMPLIED

>

Declare a Page to be added to the Style Editor dialog; the user can only work with one page at a time so be sure to make each page "complete". Examples of Pages are Theme editing; Choosing a Named Style for a WMS and so on. Pages should completly delegate their state to the layer (ie the style blackboard). Please feel free to directly attack the style blackboard as you are working with a copy. The Style Editor Dialog will refresh the layer with the real settings when the user presses OK.


  • id - Unique identifier for internal use
  • label - A human readable label used to identify the sld extension in the ui. For example, "Line", "Point", etc.
  • class - A subclass of SLDEditorPage to instantiate
  • contentType - The class of style component of interest (SLD, Style, FeatureTypeStyle, Rule, Symbolizer, etc).
  • category - if defined, this attribute will specify this element's parent id in the tree.
  • requires - if defined, the selected layer must resolve this class in order for this page to be displayed
