fileFormat

Identifier:
net.refractions.udig.catalog.ui.fileFormat

Since:
1.0

Description:
Used to teach uDig about new file extensions suitable for use in the File wizard page.

Configuration Markup:

<!ELEMENT extension ((fileService | provider)*)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT fileService EMPTY>

<!ATTLIST fileService

name          CDATA #IMPLIED

fileExtension CDATA #REQUIRED

>

Used to register a static file extension (such as "*.txt" or "*.shp) for use.


  • name - Name of the file format (may be translated into the local language)
  • fileExtension - This is the actual file extension being declaired.

<!ELEMENT provider EMPTY>

<!ATTLIST provider

class CDATA #REQUIRED

>

Dynamically generate a list of supported file formats. Used to communicate the capabilieis of an external library such as GDAL, ImageIO-Ext or GeoTools.


  • class - FormatProvider instance used to list file format extensions and translated file format names. Should only be used to sort out "optional" file formats where supporting the format is sorted out at runtime. For static listing of a file format please use the normal fileService element.

Examples:
net.refractions.udig.catalog.shp contrinutes the "*.shp" format; and provides a service connection factory capable of working with "*.shp" files.

API Information:
Implement a FormatProvider to list your supported file extensions (along with their format names).

Supplied Implementation:
Day 4 tutorial materials cover the addition of a "*.csv" file; you may review this source code in the tutorials directory.
